# 评论写手系统 使用手册

## 版本信息

- **版本**：v1.3
- **更新日期**：2026-04-17
- **主要更新**：集成产品信息知识库、优化提示工程、UI优化

---

## 系统概述

评论写手是一个基于 LLM（大语言模型）和 RAG（检索增强生成）的智能评论生成系统，通过读取数据库中真实玩家评论的风格，学习生成自然、真实、不带 AI 味的舆情评论。

系统可选集成产品信息知识库，在生成评论前先检索产品相关信息，使生成的评论更贴合产品特性。

## 环境准备

### 1. 安装依赖

```bash
cd e:/评论写手
./commenter/Scripts/pip install -r requirements.txt
```

### 2. 配置 API

在项目根目录创建或编辑 `.env` 文件：

```env
OPENAI_API_KEY=你的API密钥
OPENAI_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

> 当前默认使用 DeepSeek V3.2（DeepSeek Chat）。更换 API 时只需修改上述三项。

### 3. 启动 Milvus 数据库（Docker）

```bash
docker run -d \
  --name milvus-etcd \
  -p 2379:2379 \
  -p 2381:2381 \
  etcd:3.5.5 \
  etcd -name etcd \
  --auto-compaction-retention=1 \
  --snapshot-count=5000 \
  --data-dir /etcd

docker run -d \
  --name milvus-minio \
  -p 9000:9000 \
  -p 9001:9001 \
  minio/minio:latest \
  server /minio_data --console-address ":9001"

docker run -d \
  --name milvus \
  -p 19530:19530 \
  -p 9091:9091 \
  milvusdb/milvus:v2.4.0 \
  milvus run milvus
```

等待约 10 秒后确认服务就绪：
```bash
docker ps | grep milvus
```

### 4. 构建评论数据库（首次使用）

```bash
cd e:/评论写手
./commenter/Scripts/python build_database.py
```

此脚本会读取 `comments.jsonl` 文件，生成语义向量后存入 Milvus。数据库包含约 3 万条真实评论。

## 服务管理（PM2）

### 启动服务

```bash
cd e:/评论写手
pm2 start ecosystem.config.js
```

### 查看状态

```bash
pm2 status
```

### 查看日志

```bash
pm2 logs 评论写手服务
```

### 重启服务

```bash
pm2 restart 评论写手服务
```

### 停止服务

```bash
pm2 stop 评论写手服务
```

### 开机自启设置

由于 Windows 不支持 PM2 startup 命令，请手动设置：

1. 创建 `启动服务.bat` 的快捷方式
2. 按 `Win + R`，输入 `shell:startup`，回车
3. 将快捷方式复制到打开的文件夹中

### 启动成功后访问

- **本机访问**：`http://localhost:7860`
- **局域网访问**：`http://192.168.100.63:7860`

> 每次修改代码后需重启服务使改动生效。重启命令：`pm2 restart 评论写手服务`

### 注意事项

- 服务运行时会隐藏 Python 控制台窗口，无窗口弹出
- 确保 Milvus 数据库已启动：`docker start milvus`
- 如果端口被占用：`taskkill //F //PID <进程ID>` 后重新启动

## 界面功能说明

### 基础生成

| 参数 | 说明 |
|------|------|
| 话题 | 要生成评论的事件/话题（必填），如"新版本战斗系统"、"英雄调整" |
| 评论数量 | 1-100 条，默认 10 |
| 评论方向 | 正性向 / 中性向 / 中正性向 |
| 立场（产品） | 王者荣耀、DNF端游、金铲铲之战、无畏契约手游、洛克王国世界、王者荣耀世界、其他 |
| 事件背景 | 可选。粘贴较新的事件文章，弥补数据库无相关内容时的不足 |

### 多视角生成

在基础生成基础上，增加"视角"参数。可以模拟某个人群（如"零氪玩家"、"PVP玩家"）的视角来评价，但**立场始终是所选产品**。

| 参数 | 说明 |
|------|------|
| 视角 | 要模拟的人群，如"零氪玩家"、"PVP重度玩家"、"回归玩家" |

### "其他"产品选项

当选择"其他"时，会显示一个文本框用于输入产品名称。系统会尝试从产品信息知识库（如果已集成）中检索相关信息。

---

## 评论方向说明

- **正性向**：以赞扬、推荐为主，可以有小挑剔但整体基调正面
- **中性向**：以陈述事实和客观描述为主，克制表达，不偏向任何一方
- **中正性向**：偏正面但有所保留，不像正性向那么热情

---

## 产品立场说明

系统支持多产品立场切换，不同产品使用对应数据库中的评论风格进行 RAG 检索：

| 产品 | 说明 |
|------|------|
| 王者荣耀 | 默认，支持皮肤、英雄、赛事等舆情评论 |
| DNF端游 | 支持地下城与勇士端游相关内容评论 |
| 金铲铲之战 | 支持英雄联盟自走棋相关内容评论 |
| 无畏契约手游 | 支持VALORANT手游相关内容评论 |
| 洛克王国世界 | 支持洛克王国IP新作相关内容评论 |
| 王者荣耀世界 | 支持王者荣耀开放世界版本相关内容评论 |
| 其他 | 支持未列出产品，可手动输入产品名称 |

## 产品信息知识库（可选）

如需系统更了解产品特性，可集成产品信息知识库。详细说明请参考 `E:\产品信息知识库\README.md`。

集成后，系统在生成评论前会先检索产品相关报道，将产品背景信息融入生成prompt，使评论更贴合产品实际特性。

## 事件背景功能

当话题涉及**非常新的事件**（数据库中尚未收录）时，RAG 检索可能找不到相关评论。此时可在"事件背景"文本框中粘贴事件相关文章或详细描述，LLM 将直接基于这段内容生成评论，而不依赖数据库检索结果。

## 提示工程设计原则

系统生成的评论经过多重优化，以消除 AI 味：

### 基础规则
1. **禁止感叹号**：`！` 和 `!` 全部去除
2. **禁用连接词**：严禁使用"首先/其次"、"总的来说"、"不过话说回来"等
3. **拒绝找补**：表达完核心观点后立刻结束，不加平衡性话术

### 表达风格
4. **碎片化表达**：短句、无主语、口语化，模拟真实随手评论
5. **打破逻辑链**：不追求完整论证结构，思维可以跳跃
6. **禁止三明治结构**：先夸后批再总结是AI典型特征，必须禁止

### 长度要求
7. **短评论（20-40字）**：必须有实质内容，不得少于20字
8. **禁用敷衍词汇**："还行"、"凑合"、"一般般"等空洞套话严格禁止
9. **长评论（75字以上）**：思维跳跃、话题漂移，不需要完整结尾

### 细节要求
10. **表情符号**：约三分之一评论可带表情符号，如[大哭][裂开][doge]等
11. **自然融入细节**：可包含具体数据增强真实感，但避免游戏专属细节
12. **禁止说明书式植入**：提到游戏元素要像随口提及，不做攻略推荐

## 常见问题

### 1. 服务启动报错"端口已被占用"
```bash
# 查找占用端口的进程
netstat -ano | findstr 7860

# 终止旧进程后重新启动
taskkill //F //PID <进程ID>
./commenter/Scripts/python run.py
```

### 2. 生成失败"API 配置错误"
- 检查 `.env` 文件中的 `OPENAI_API_KEY` 是否正确
- 检查 `OPENAI_BASE_URL` 是否为 `https://api.deepseek.com/v1`
- 确认 API 余额充足

### 3. 生成数量少于请求数量
- 可能是 max_tokens 截断，调整 `comment_generator.py` 中的 `max_tokens` 值
- 也可能是模型输出被截断，可查看后台日志中的解析结果

### 4. 数据库为空
```bash
# 重新运行数据库构建
./commenter/Scripts/python build_database.py
```

### 5. 修改代码后 UI 没有更新
- 彻底关闭浏览器后重新打开
- 清除浏览器缓存（Ctrl+Shift+R）
- 确认服务器进程已重启

### 6. 产品知识库检索失败
- 确保产品信息知识库的 Milvus 服务已启动
- 确保 `E:/产品信息知识库` 路径存在且包含必要文件
- 检查终端输出中的 "产品知识库检索失败" 错误信息
