# 评论写手系统 使用手册

## 系统概述

评论写手是一个基于 LLM（大语言模型）和 RAG（检索增强生成）的智能评论生成系统，通过读取数据库中真实玩家评论的风格，学习生成自然、真实、不带 AI 味的舆情评论。

---

## 环境准备

### 1. 安装依赖

```bash
cd e:/评论写手
./commenter/Scripts/pip install -r requirements.txt
```

### 2. 配置 API

在项目根目录创建或编辑 `.env` 文件：

```env
OPENAI_API_KEY=你的API密钥
OPENAI_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

> 当前默认使用 DeepSeek V3.2（DeepSeek Chat）。更换 API 时只需修改上述三项。

### 3. 启动 Milvus 数据库（Docker）

```bash
docker run -d \
  --name milvus-etcd \
  -p 2379:2379 \
  -p 2381:2381 \
  etcd:3.5.5 \
  etcd -name etcd \
  --auto-compaction-retention=1 \
  --snapshot-count=5000 \
  --data-dir /etcd

docker run -d \
  --name milvus-minio \
  -p 9000:9000 \
  -p 9001:9001 \
  minio/minio:latest \
  server /minio_data --console-address ":9001"

docker run -d \
  --name milvus \
  -p 19530:19530 \
  -p 9091:9091 \
  milvusdb/milvus:v2.4.0 \
  milvus run milvus
```

等待约 10 秒后确认服务就绪：
```bash
docker ps | grep milvus
```

### 4. 构建评论数据库（首次使用）

```bash
cd e:/评论写手
./commenter/Scripts/python build_database.py
```

此脚本会读取 `comments.jsonl` 文件，生成语义向量后存入 Milvus。数据库包含约 3 万条真实评论。

## 界面功能说明

### 基础生成

| 参数 | 说明 |
|------|------|
| 话题 | 要生成评论的事件/话题（必填），如"孙策新皮肤"、"英雄调整" |
| 评论数量 | 1-100 条，默认 10 |
| 评论方向 | 正性向 / 中性向 / 中正性向 |
| 立场（产品） | 当前支持：王者荣耀、原神、三角洲行动、洛克王国世界、王者荣耀世界 |
| 事件背景 | 可选。粘贴较新的事件文章，弥补数据库无相关内容时的不足 |

### 多视角生成

在基础生成基础上，增加"视角"参数。可以模拟某个人群（如"原神玩家"、"FPS 玩家"）的视角来评价，但**立场始终是所选产品**。

| 参数 | 说明 |
|------|------|
| 视角 | 要模拟的人群，如"原神玩家"、"王者荣耀主播"、"FPS 重度玩家" |

---

## 评论方向说明

- **正性向**：以赞扬、推荐为主，可以有小挑剔但整体基调正面
- **中性向**：以陈述事实和客观描述为主，克制表达，不偏向任何一方
- **中正性向**：偏正面但有所保留，不像正性向那么热情

---

## 产品立场说明

系统支持多产品立场切换，不同产品使用对应数据库中的评论风格进行 RAG 检索：

| 产品 | 说明 |
|------|------|
| 王者荣耀 | 默认，支持皮肤、英雄、赛事等舆情评论 |
| 原神 | 支持原神相关内容评论 |
| 三角洲行动 | 支持三角洲行动相关内容评论 |
| 洛克王国世界 | 支持洛克王国世界相关内容评论 |
| 王者荣耀世界 | 支持王者荣耀世界（王者荣耀IP新游戏）相关内容评论 |

---

## 事件背景功能

当话题涉及**非常新的事件**（数据库中尚未收录）时，RAG 检索可能找不到相关评论。此时可在"事件背景"文本框中粘贴事件相关文章或详细描述，LLM 将直接基于这段内容生成评论，而不依赖数据库检索结果。

---

## 提示工程设计原则

系统生成的评论经过多重优化，以消除 AI 味：

1. **禁止感叹号**：`！` 和 `!` 全部去除
2. **克制表达**：禁止"绝了"、"太棒了"、"yyds"等情绪化夸张词
3. **碎片化表达**：短句、无主语、口语化，模拟真实随手评论
4. **打破逻辑链**：不追求完整论证结构，思维可以跳跃
5. **中性正向长评**：长评论不整段夸赞，而是先中性描述或略带挑剔，最后1-2句正向
6. **拒绝找补**：表达完核心观点后立刻结束，不加平衡性话术

## 常见问题

### 1. 服务启动报错"端口已被占用"
```bash
# 查找占用端口的进程
netstat -ano | grep 7860

# 终止旧进程后重新启动
taskkill //F //PID <进程ID>
./commenter/Scripts/python run.py
```

### 2. 生成失败"API 配置错误"
- 检查 `.env` 文件中的 `OPENAI_API_KEY` 是否正确
- 检查 `OPENAI_BASE_URL` 是否为 `https://api.deepseek.com/v1`
- 确认 API 余额充足

### 3. 生成数量少于请求数量
- 可能是 max_tokens 截断，调整 `comment_generator.py` 中的 `max_tokens` 值
- 也可能是模型输出被截断，可查看后台日志中的解析结果

### 4. 数据库为空
```bash
# 重新运行数据库构建
./commenter/Scripts/python build_database.py
```

### 5. 修改代码后 UI 没有更新
- 彻底关闭浏览器后重新打开
- 清除浏览器缓存（Ctrl+Shift+R）
- 确认服务器进程已重启
